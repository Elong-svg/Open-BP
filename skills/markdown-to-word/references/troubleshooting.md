## 常见问题与解决方案

### 问题 1：图片无法插入

**症状**：
- Word 文档中没有图片
- 控制台提示 `[IMG] 跳过：图片不存在 xxx`

**根本原因与解决**：

| 原因 | 解决方案 |
|------|---------|
| 路径错误 | 1) 确认文件存在 2) 检查大小写 3) 用绝对路径测试 |
| `../` 路径被拒 | v3.0.0 已自动处理：尝试去除 `../` 前缀后重新解析 |
| 网络图片下载失败 | 检查网络，确认 URL 可访问（脚本用 requests 下载，10s 超时） |
| 缺 requests 库 | `pip install requests` |
| 图片格式不支持 | 确认是 PNG/JPG/GIF/WebP 之一 |

**验证方法**：
```
[INFO] 统计：N 张图片，...   ← N 应等于 md 中的图片数量
```

---

### 问题 2：超链接显示为普通文本

**症状**：链接没有蓝色下划线，点击不跳转

**解决方案**（v3.0.0+）：
- 外部链接（http/https）：自动添加超链接关系
- 锚点链接（#xxx）：降级为加粗文字
- 文件协议（file://）：安全检查可能会拒绝

---

### 问题 3：代码块显示为正文

**症状**：` ``` python ` 包起来的代码没有浅灰背景

**解决方案**（v3.0.0+）：
- 确认使用 ` ``` ` 围栏代码块
- 脚本默认已启用 fenced_code 扩展
- 正常情况会显示浅灰背景 + 等宽字体

---

### 问题 4：嵌套列表错位

**症状**：多级嵌套列表缩进混乱

**解决方案**（v3.0.0+）：
- 脚本使用 sane_lists 扩展，能正确处理任意层级
- 确认列表前用空行隔开

---

### 问题 5：emoji 导致转换失败

**症状**：
```
UnicodeEncodeError: 'gbk' codec can't encode character '\U0001f4d6'
```

**解决方案**（v3.0.0+）：
- 脚本已移除所有 emoji 标记，改用纯文本 `[INFO]` `[SUCCESS]`
- Windows PowerShell 仍可能因 GBK 编码报错 → 用 Windows Terminal 或 VSCode

---

### 问题 6：表格没有样式

**症状**：表格显示但表头是白的、没有条纹

**根本原因**：
- 模板 XML 文件被改动
- Python-docx 版本过低

**解决方案**：
```bash
pip install --upgrade python-docx
# 检查 XML 配置
cat assets/styles/corporate_styles.xml | grep "table_header"
```

---

### 问题 7：中文乱码

**症状**：Word 打开后中文变成问号或方框

**解决方案**：
1. 确认 Markdown 文件是 UTF-8 编码
2. 确认模板中的 font_family 配置正确
3. Word 打开时选择"UTF-8"编码

---

### 问题 8：路径安全检查拒绝合法路径

**症状**：
```
[IMG] 跳过：图片不存在 xxx
```

**常见原因**：
- v3.0.0 之前的版本不支持 `../` 路径
- 路径指向系统目录（C:\Windows 等）

**解决方案**：
- 升级到 v3.0.0+
- 把图片复制到 markdown 同级目录
- 使用绝对路径
